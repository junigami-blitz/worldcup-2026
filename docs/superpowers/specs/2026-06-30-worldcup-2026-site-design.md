## 1. 目的とスコープ

2026 FIFAワールドカップ（カナダ・メキシコ・USA共催、48チーム / 12グループ / 全104試合の新フォーマット）を、わかりやすく解説する**公開Webサイト**を作る。グループリーグ・決勝トーナメントの組み合わせと結果、各種ランキング、主要ニュース、そして可能な範囲で**YouTubeの試合ハイライト動画**を掲載し、これらを**無料の手段で自動更新**する。

開催期間中（今まさに進行中）のリアルタイム反映を前提とする。

### やること
- グループステージ（12グループ）の順位表・日程・結果
- 決勝トーナメント（32強→決勝）のブラケットと結果
- ランキング（得点王・アシスト・警告/退場・クリーンシート・チームスタッツ・FIFAランキング等、取得できるものは全部）
- 主要ニュース（Google News RSS 日本語）
- 試合ハイライト動画（YouTube、公式優先 + 指定チャンネル群フォールバック）
- 上記の無料・自動更新

### やらないこと（YAGNI）
- 分単位のライブスコア中継（無料枠の制約上）
- ユーザー登録・コメント・予想投票などのインタラクティブ機能
- 有料データAPI・有料ホスティング
- ネイティブアプリ

## 2. アーキテクチャ

**サーバーを持たない静的サイト + ビルド時データ流し込み**方式。SPAにはしない。

```
[openfootball / football-data.org / YouTube Data API / Google News RSS]
        │  Pythonスクリプトがビルド時に取得
        ▼
   data/*.json （リポジトリにコミット）
        │  GitHub Actions（cron）が取得→コミット→Pagesへデプロイ
        ▼
   静的サイト（生成済みHTML/CSS/JS）= GitHub Pages で公開
        │  ブラウザは完成済みページを表示するだけ（速い・SEOに強い）
        ▼
       閲覧者
```

- **ホスティング**: GitHub Pages（公開リポジトリ、無料）
- **自動化**: GitHub Actions（公開リポジトリは無料）
- **シークレット**: APIキーは GitHub Secrets に格納（サイト側に露出しない）
- **非SPA**: ブラウザでのAPI直叩きはしない。データ取得はすべてActions上（ビルド時）で完結

### なぜこの方式か
- サーバーレス＝落ちない・課金事故ゼロ
- 既存資産（Pythonスクレイピング、`scrape_api.py`）が活きる
- cronで放置運用（無料×自動×低メンテ）が成立

## 3. データソース（すべて無料）

| 用途 | ソース | 形式 / 認証 | 備考 |
|---|---|---|---|
| 大会の骨格（グループ分け・全104試合・代表メンバー・スタジアム・チーム） | [openfootball/worldcup.json](https://github.com/openfootball/worldcup.json) の `2026/` | GitHub raw JSON / **キー不要・パブリックドメイン** | 構造が安定。スクレイピングより壊れにくい。1次ソース |
| 試合の確定結果・順位 | football-data.org 無料枠 + openfootball | REST / APIキー（無料登録） | 無料枠は**ライブスコア非対応**。確定結果が数分〜十数分遅れで反映 |
| ハイライト動画 | YouTube Data API v3 | REST / APIキー（無料枠1万ユニット/日） | 検索100ユニット/回。1日約100検索可。結果はキャッシュ |
| ニュース | Google News RSS（日本語） | RSS / 認証不要 | 「ワールドカップ2026」等のクエリ |

### 既知の制約・正直な注記
- football-data 無料枠は分単位ライブスコア・詳細スタッツ・オッズ非対応。2時間おき運用なら遅延は実用上問題なし。
- openfootball はコミュニティ更新のため、結果反映にラグが出る場合がある。football-data と併用して鮮度を補う。
- 取得できないスタッツ項目（アシスト等がデータに無い場合）は、サイト上で**自動的に非表示**にする（壊さない）。

## 4. コンポーネント（各ファイル＝単一責務）

| ファイル | 責務 | 実行タイミング | 依存 |
|---|---|---|---|
| `scripts/fetch_structure.py` | openfootballから骨格を取得 → `data/structure.json`（グループ・日程・代表・スタジアム） | 1日1回 | openfootball |
| `scripts/fetch_results.py` | football-data + openfootball から確定結果・順位 → `data/results.json` | 試合時間帯のみ | football-data, openfootball |
| `scripts/compute_rankings.py` | 結果から順位表・得点王・アシスト・警告/退場・クリーンシート等を自前集計 → `data/rankings.json` | 結果取得後 | results.json |
| `scripts/fetch_youtube.py` | 終了済み試合ごとにハイライト検索（公式優先→指定チャンネル群）→ `data/highlights.json` | 試合時間帯のみ | YouTube API, results.json |
| `scripts/fetch_news.py` | Google News RSS → `data/news.json` | 1日数回 | RSS |
| `scripts/schedule_gate.py` | 今が試合時間帯か判定。無ければ即終了 | 各実行の先頭 | structure.json |
| `scripts/build_site.py` | data/*.json を読み、HTMLテンプレートに流し込んで `site/` に静的HTML生成 | データ更新後 | data/*.json, templates/ |
| `templates/` + `site/assets/` | デザイン（HTML/CSS/JS、v2スタイル） | — | — |

### 堅牢性の共通ルール
- 各fetcherは**一時ファイルに書いてからアトミック差し替え**。取得失敗時は**前回の正常JSONを保持**（サイトが空白にならない）。
- YouTubeは解決済みハイライトをキャッシュし、**未解決の終了済み試合だけ**を検索（クォータ節約）。
- football-data はレート制限（10req/分）に合わせてスロットリング。

## 5. 自動更新スケジュール

GitHub Actions の cron は固定時刻のため、**「広めのcron × 賢いゲート」**で「試合時間帯だけ約2時間おき、無試合帯は回さない」を実現する。

- `schedule_gate.py` が `structure.json` の試合スケジュールを読み、現在時刻が**いずれかの試合のキックオフ〜終了後（kickoff 〜 +3h）の窓**に入っているか判定。
  - 窓内 → `fetch_results` → `compute_rankings` → `fetch_youtube` → `fetch_news` → `build_site` → コミット&デプロイ
  - 窓外 → 数秒で終了（実質的に試合時間帯のみ稼働）
- 骨格更新（`fetch_structure`）は1日1回の別ワークフロー。
- ニュースは試合の有無に関わらず1日数回更新してもよい（任意）。
- 大会終了後はゲートが常にfalseになり、自動的に静穏化する。

## 6. サイト構成（ページ）

非SPA。各ページは生成済み静的HTML。

1. **トップ**: 今日の試合 / 直近結果 / 注目ハイライト / 最新ニュースのダッシュボード
2. **グループステージ**: 12グループの順位表 + 日程 + 結果（各試合カードに得点者・ハイライトリンク）
3. **決勝トーナメント**: 32強→決勝のブラケット図（48チーム新フォーマット対応）+ 結果
4. **ランキング**: 得点王・アシスト・警告/退場・クリーンシート・チームスタッツ・FIFAランキング（無データ項目は自動非表示）
5. **ニュース**: Google News RSS フィード

## 7. デザイン方針（確定: 日テレ風 v2「作り込んだ放送局プロダクト」）

参考: 日本テレビ FIFAワールドカップ2026特設サイト。そのDNAを継ぎつつ、**AI感（テンプレ感）を排した質感**にする。

### カラー
- ベース: 白 `#ffffff` / 文字 `#1c1c1e`
- メイン: ロイヤルブルー `#0162b1`（順位・リンク・突破ライン）
- アクセント: ビビッドレッド `#d50101`（キッカー、アクティブタブ下線）※多用しない
- サブグレー: `#6b6b70`（補助情報）/ ヘアライン `#ececef` `#f1f1f3`

### タイポグラフィ
- 日本語: Noto Sans JP（500/700/900）
- 英数・数字・ラベル: **Oswald**（コンデンス、`font-variant-numeric: tabular-nums` で等幅数字）→ スコアやスタッツにスポーツ紙面の質感

### スタイル原則（"AI感を薄める"の具体)
- 見出しは絵文字を使わず、**レタースペースした小さなキッカー**（例: `GROUP E — MATCHDAY 2`）
- ナビは全部ピル型をやめ、**テキストタブ + 下線インジケーター**（アクティブのみ赤線）
- 影は最小限。**ヘアラインで情報を整理**（多重シャドウ・ドット背景・グラデ過剰を避ける）
- 国旗は絵文字でなく**CSS/画像の小さな国旗チップ**で精度を出す
- ハイライトは大きな赤枠ボタンでなく**控えめなテキストリンク（▷）**
- 順位表の突破ラインは**細い色の左罫**で表現
- レスポンシブ必須（モバイルファースト）
